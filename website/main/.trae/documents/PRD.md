# 服务商管理系统 - 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目名称
**ProviderHub** - 服务商管理与展示平台

### 1.2 项目定位
一个面向B2B/B2C场景的服务商管理平台，连接平台运营方、入驻服务商与终端用户。平台运营方通过后台管理系统对服务商进行全生命周期管理；终端用户通过前台浏览、搜索、对比并联系服务商。

### 1.3 目标用户
- **平台管理员**：负责服务商审核、分类、状态管理、运营决策
- **服务商**：申请入驻、维护资料、查看统计数据
- **终端用户**：浏览、搜索、筛选服务商信息

### 1.4 核心价值
- 高效的服务商准入与审核流程
- 清晰的服务商分类与发现机制
- 透明的服务商数据统计与运营分析
- 安全可靠的数据传输与访问控制

## 2. 功能需求

### 2.1 后台管理系统（Admin Console）

#### 2.1.1 服务商管理
- **CRUD操作**：添加、编辑、查询、删除服务商信息
- **服务商信息字段**：
  - 基础信息：名称、统一社会信用代码、Logo、简介、官网
  - 联系信息：联系人、联系电话、邮箱、地址
  - 业务信息：服务范围、价格区间、服务区域、合作模式
  - 资质材料：营业执照、行业许可证、相关证书（图片/PDF）
  - 评分体系：服务评分、响应速度、成交率
- **批量操作**：批量启用/禁用/删除
- **高级筛选**：按状态、分类、地区、入驻时间、评分多维度筛选

#### 2.1.2 分类管理
- **多级分类**：支持两级分类（一级分类 + 二级分类）
- **分类CRUD**：增删改查
- **分类属性**：名称、图标、父分类、排序、显示状态
- **服务商标签**：分类下的标签管理

#### 2.1.3 资质审核流程
- **审核状态**：待审核、审核中、已通过、未通过、已撤回
- **审核流程**：
  1. 服务商提交入驻申请
  2. 平台初审（材料完整性）
  3. 平台复审（资质真实性）
  4. 审核结果通知
  5. 服务商可对未通过结果申诉
- **审核记录**：完整审核历史、审核人、审核意见
- **批量审核**：支持批量通过/拒绝

#### 2.1.4 状态管理
- **服务商状态**：启用、禁用、待审核、已过期
- **状态切换日志**：完整的状态变更记录
- **禁用原因**：禁用时必填原因

#### 2.1.5 数据统计与分析
- **总览仪表盘**：
  - 服务商总数、各状态分布
  - 今日新增、本周新增、本月新增
  - 审核待办数量
  - 访问量、咨询量趋势
- **服务商分析**：
  - TOP 10 服务商（按评分/访问量/成交量）
  - 分类分布饼图
  - 地区分布地图
  - 入驻趋势折线图
- **数据导出**：支持Excel/CSV格式导出

### 2.2 前端用户系统

#### 2.2.1 服务商浏览
- **首页**：精选推荐服务商、分类入口、最新入驻
- **服务商列表页**：分页、排序、筛选
- **服务商详情页**：完整资料、资质展示、用户评价、联系方式
- **对比功能**：支持多家服务商并排对比

#### 2.2.2 搜索功能
- **关键词搜索**：服务商名称、简介、标签
- **高级搜索**：按分类、地区、价格、评分组合搜索
- **搜索建议**：输入时自动补全
- **搜索历史**：本地记录最近搜索

#### 2.2.3 筛选与排序
- **筛选维度**：分类、地区、价格区间、评分、合作模式
- **排序方式**：综合、评分最高、价格最低、距离最近

#### 2.2.4 用户交互
- **收藏功能**：登录用户可收藏服务商
- **联系咨询**：点击联系记录咨询意向
- **评价功能**：合作后可评价

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 ≤ 2秒
- API响应时间 ≤ 200ms
- 支持1000+并发用户
- 数据库查询优化：核心列表查询 ≤ 100ms

### 3.2 安全要求
- JWT身份认证，Token有效期24小时
- 密码bcrypt加密（cost=12）
- 敏感操作二次验证
- API请求限流：100次/分钟/IP
- 防SQL注入：GORM参数化查询
- 防XSS：输入输出转义
- HTTPS传输加密
- 敏感字段加密存储（身份证、银行卡）

### 3.3 可用性要求
- 系统可用性 ≥ 99.5%
- 关键操作成功率 ≥ 99.9%
- 错误处理：友好错误提示

### 3.4 可扩展性
- 模块化设计：前后端分离
- 接口RESTful标准化
- 支持水平扩展
- 配置外部化，便于多环境部署

### 3.5 可维护性
- 完整的日志系统
- 健康检查接口
- 操作审计日志
- 代码注释完整

## 4. 用户体验要求
- 界面响应式设计：PC端、平板、手机自适应
- 加载状态、错误状态、空状态完整
- 操作流程：≤3步完成核心操作
- 视觉风格：现代简洁、专业可信

## 5. 技术约束
- **前端**：原生 HTML5 + CSS3 + JavaScript（ES6+）
- **后端**：Go + Gin + GORM
- **数据库**：SQLite（开发）/ MySQL（生产）
- **部署**：静态文件 + Go 二进制

## 6. 验收标准
- 后台所有CRUD功能正常
- 前端所有页面正常浏览
- 数据统计图表正确展示
- API响应符合性能要求
- 关键路径安全性验证通过
